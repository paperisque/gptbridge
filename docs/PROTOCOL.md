# Контракт WS — VoiceBridge ⇄ Firefox-расширение и сетевые клиенты

Единый источник правды для C#-сервера, Firefox-расширения и сетевого клиента. Меняем формат — правим **этот файл**, потом все стороны (`WsServer.OnMessageAsync` + `WsClient.OnMessage` + обработчики в `content.js`/`background.js`).

> **Этот контракт — для Firefox-версии.** Автономная версия на WebView2 (`poc-webview2/`,
> GPT Grabber) WebSocket НЕ использует: она управляет встроенной страницей ChatGPT напрямую,
> инъекцией JS из C# (`CoreWebView2.ExecuteScriptAsync`). См. раздел «WebView2-версия» внизу
> и [CLAUDE.md](../CLAUDE.md).

## Транспорт

- WebSocket поверх HTTP, без TLS.
- Адрес: `ws://<host>:17890/`, где `host`:
  - `localhost` — по умолчанию (всё на одной машине, без прав администратора);
  - `+` / IP машины-сервера — для доступа по сети (нужен разовый `netsh http add urlacl` и правило брандмауэра, см. README).
- Кодировка: UTF-8, текстовые WS-фреймы.

## Топология (сервер — хаб)

Сервер — центральный узел. К нему подключаются:

- **Firefox-расширение** (движок диктовки) — `background.js`. Кликает кнопки диктовки в ChatGPT, отдаёт распознанный текст.
- **Сетевые клиенты-контроллеры** — `VoiceBridge --connect <host>` на других машинах (`WsClient`/`ClientApp`). Ловят свой Ctrl+Win, захватывают своё активное окно, вставляют текст у себя.

Плюс **локальный контроллер** живёт в самом сервере (`ServerApp`) и по WS не ходит.

Сервер различает соединения по **роли**, определяемой из первого `hello`:
`payload` содержит `controller` → контроллер; иначе → Firefox.

**Владелец сессии.** Диктовка — один общий ресурс (один Firefox). Кто запустил сессию (локальный контроллер или конкретный сетевой клиент), тому сервер и вернёт распознанный текст. Пока идёт сессия одного владельца, запросы старта от других игнорируются («занято»).

## Формат сообщения

Каждое сообщение — один JSON-объект:

```json
{ "type": "<string>", "payload": "<string>" }
```

Поля всегда строки. `payload` для сообщений без данных — пустая строка `""`.

## Сообщения: Firefox-расширение → сервер

| type        | payload                    | смысл                                                                 |
|-------------|----------------------------|-----------------------------------------------------------------------|
| `hello`     | произвольная строка (инфо) | представление; payload БЕЗ слова `controller` ⇒ роль Firefox          |
| `text`      | полный текст композера     | текст в `#prompt-textarea` стабилизировался — забери и доставь владельцу |
| `recording` | `""`                       | запись реально началась (появилась кнопка `Submit`/`Cancel dictation`) |
| `ready`     | `""`                       | таб ChatGPT открыт и готов к диктовке (есть кнопка `Start dictation`) — ответ на `ensureTab` |
| `empty`     | `""`                       | распознавание после `stop` дало пустоту (в записи была тишина) — текста не будет |

- На `text` сервер сохраняет payload как «последний текст», отвечает `ack` и (если вооружён СТОПом) доставляет владельцу сессии.
- На `recording` сервер возвращает фокус в окно, активное до старта диктовки (на машине-сервере).
- `ready` нужен серверу для бесшовного старта: если ждали поднятия Firefox/открытия таба под запись — по `ready` начинаем запись; если это прогрев (старт сервера/реконнект) — просто фиксируем готовность.

## Сообщения: сервер → Firefox-расширение

| type        | payload               | смысл                                                       |
|-------------|-----------------------|-------------------------------------------------------------|
| `hello`     | `"VoiceBridge ready"` | приветствие сразу после подключения                         |
| `ack`       | число (длина текста)  | подтверждение приёма `text`                                 |
| `ensureTab` | подсказки (см. ниже)  | убедиться, что таб ChatGPT открыт и готов; открыть, если нет; ответить `ready` |
| `mic`       | `""`                  | кликнуть «Start dictation» (старт)                          |
| `stop`      | `""`                  | кликнуть «Submit dictation» (финал)                         |
| `cancel`    | `""`                  | кликнуть «Cancel dictation» — отбой без распознавания (захват не стартовал) |
| `clear`     | `""`                  | очистить композер ChatGPT (после доставки текста)           |

`ensureTab`/`mic`/`stop`/`cancel`/`clear` шлются адресно — только соединениям-Firefox (`SendToFirefox`).

**payload `ensureTab`** — подсказки расширению (строка, флаги через запятую):
- `start` — подготовка под запись: если таба нет, открыть его **активным** (фоновые табы Firefox грузит с пониженным приоритетом — пассивный открывается заметно дольше);
- `warmup` — просто прогрев: новый таб открывать пассивно (`active:false`), пользователя не дёргать;
- `,cold` — Firefox только что запущен сервером: его стартовый таб ChatGPT может ещё грузиться и не быть видимым в `tabs.query` — расширение дольше ждёт его появления, прежде чем создавать второй (иначе дубль).

Пустой payload трактуется как `warmup` (обратная совместимость).

Сервер шлёт `ensureTab`: при подключении расширения (прогрев) и перед стартом (если расширение на связи). Если Firefox вообще не запущен, сервер сам поднимает его — БЕЗ URL (передача URL давала дубль таба при восстановлении сессии): табы восстановит сессия, а таб ChatGPT найдёт/создаст расширение по `ensureTab`; «ленивый» (discarded) таб из восстановленной сессии расширение предварительно будит. Внутри расширения готовность таба определяется внутренним пингом `content.js` (`checkReady` — НЕ часть WS-контракта).

## Сообщения: сетевой клиент → сервер

| type    | payload        | смысл                                                          |
|---------|----------------|----------------------------------------------------------------|
| `hello` | `"controller"` | представление; payload со словом `controller` ⇒ роль контроллер |
| `start` | `""`           | локальный Ctrl+Win из `Idle` — клиент просит старт диктовки     |
| `stop`  | `""`           | локальный Ctrl+Win из `Recording` — клиент просит финал         |

Клиент сам ловит хоткей, по `stop` захватывает СВОЁ активное окно и помнит вариант «+буфер» (Ctrl+Win+Y) — серверу это знать не нужно.

## Сообщения: сервер → сетевой клиент

| type     | payload                 | смысл                                                       |
|----------|-------------------------|-------------------------------------------------------------|
| `hello`  | `"VoiceBridge ready"`   | приветствие сразу после подключения                         |
| `inject` | текст диктовки          | этот текст вставить в окно, захваченное клиентом по `stop`   |

`inject` шлётся адресно владельцу сессии (`SendToController` по Id соединения).

## Поток (happy path)

Управление — клавишей **Ctrl+Win** (ловится низкоуровневым хуком: в сервере — для локального контроллера, в клиенте — для сетевого). Вариант **Ctrl+Win+«Y»** — то же завершение, но текст остаётся в буфере обмена (обрабатывается на стороне того контроллера, что вставляет).

1. Firefox-расширение подключается к серверу, шлёт `hello` (роль Firefox) → сервер шлёт `ensureTab` (прогрев: открыть таб ChatGPT, если нет). Сетевые клиенты (если есть) подключаются и шлют `hello` с `controller`.
2. **Ctrl+Win** (`Idle`) у контроллера → сервер запоминает владельца сессии и активное окно (на машине-сервере) и переходит в `Preparing`:
   - расширение на связи → `ensureTab` → расширение открывает/проверяет таб и отвечает `ready`;
   - Firefox не запущен → сервер сам поднимает его (`firefox https://chatgpt.com/`); расширение подключится и пришлёт `ready`.
   Получив `ready`, сервер **на мгновение выносит окно Firefox на передний план** (иначе FF не стартует захват микрофона) → `mic` → расширение кликает «Start dictation». Состояние → `Recording`. Если `ready` не пришёл за таймаут — старт отменяется (`Idle`).
   - локальный контроллер: жмёт хоткей сам сервер;
   - сетевой клиент: шлёт серверу `start`.
3. Расширение, увидев индикатор записи, шлёт `recording` → сервер **возвращает фокус** в запомненное окно (страховка по таймеру `Config.RecordingFocusTimeoutMs`).
4. Пользователь диктует (таб ChatGPT в фоне — запись уже идёт).
5. **Ctrl+Win** (`Recording`) у того же контроллера → `stop` → расширение кликает «Submit dictation». Окно-цель: для локального владельца захватывает сервер; для сетевого — сам клиент у себя.
6. ChatGPT транскрибирует → текст появляется в `#prompt-textarea` → content script шлёт `{type:"text", ...}`.
7. Сервер вооружён завершением → доставляет текст владельцу:
   - локальный владелец → вставка на месте (Ctrl+V);
   - сетевой владелец → `inject` ему по WS, клиент вставляет в своё окно.
   Затем сервер шлёт `clear` Firefox (поле очищается под новый раунд).
   - Если был **Ctrl+Win+«Y»** — текст дополнительно остаётся в буфере обмена той машины, где он вставлен.

## Кнопки ChatGPT (для content script)

Якорь — стабильная `<form>` (класс содержит `composer`); контролы пересоздаются,
поэтому ищутся заново при каждом действии. Во время записи `#prompt-textarea` исчезает.

| действие              | селектор                                  |
|-----------------------|-------------------------------------------|
| микрофон (старт)      | `button[aria-label="Start dictation"]`    |
| птичка (финал)        | `button[aria-label="Submit dictation"]`   |
| отмена                | `button[aria-label="Cancel dictation"]`   |

Наличие кнопки `Submit`/`Cancel dictation` = запись идёт (индикатор для сигнала `recording`).
Кнопка отправки промпта НЕ кликается, но служит индикатором исхода распознавания:
появилась после диктовки ⇔ распознался текст; при тишине её нет (вместо неё — кнопка
голосового режима) → расширение шлёт `empty`. Ищется цепочкой `#composer-submit-button`
→ `button[data-testid="send-button"]` → `button[aria-label="Send prompt"]`.

## Сбои и отбой (не happy path)

- **Захват не стартовал** (нет звука/микрофона): сигнал `recording` так и не пришёл. СТОП от владельца в этом случае НЕ шлёт `stop` (Submit завис бы на распознавании пустоты), а шлёт **`cancel`** — расширение кликает «Cancel dictation», сервер сразу возвращается в `Idle` без ожидания текста.
- **Запись шла, но в ней тишина**: после `stop` ChatGPT распознаёт пустоту, композер остаётся пустым, `text` не будет. Расширение следит за исходом (UI диктовки закрылся + композер пуст ~2 c) и шлёт **`empty`** — сервер немедленно разоружается, индикатор «пусто».
- **Текст после `stop` не пришёл** за `Config.InjectTimeoutMs` (страховка, если и `empty` потерялся): сервер разоружается (поздний `text` игнорируется), индикатор показывает ошибку.
- **Повторный запуск Firefox** подавляется launch-grace (`Config.FirefoxLaunchGraceMs`): прогрев на старте сервера и старт записи не открывают два окна браузера.

## WebView2-версия (GPT Grabber): без WS, прямая инъекция JS

Автономное приложение `poc-webview2/` встраивает ChatGPT через WebView2 и управляет
страницей не по WebSocket, а вызывая JS прямо в ней (`ExecuteScriptAsync`) в нужные
моменты. Эквивалент «сообщений» здесь — набор скриптов в `MainForm.cs`:

| операция               | что делает                                                            |
|------------------------|-----------------------------------------------------------------------|
| `StartReadyScript`     | есть ли кнопка «Start dictation» (готовность к старту)                 |
| `ClickStartScript`     | клик «Start dictation» (старт записи)                                  |
| `DictationStateScript` | идёт ли запись (есть `Submit`/`Cancel dictation` или исчез композер)   |
| `SubmitScript`         | клик «Submit dictation» (финал → распознавание)                       |
| `CancelScript`         | клик «Cancel dictation» (отбой без распознавания)                     |
| `ComposerReadScript`   | прочитать распознанный текст из `#prompt-textarea`                     |
| `ClearComposerScript`  | очистить композер (фокус + выделить всё + `execCommand('delete')`)     |
| `HideExtrasScript`     | вкрутить в `<head>` CSS, прячущий лишние блоки-соседи `#thread-bottom` |

Селекторы кнопок ChatGPT — те же, что у Firefox-версии (см. таблицу «Кнопки ChatGPT»), и
так же матчатся мультиязычно (EN/DE/RU): aria-label локализован. Захват окна-цели и Ctrl+V
приложение делает локально само (как локальный контроллер у сервера) — сети тут нет, поэтому
ни `hello`/`start`/`stop`/`inject`, ни ролей/владельца сессии в этой версии нет.
