# Контракт WS — VoiceBridge ⇄ расширение

Единый источник правды для C#-сервера и Firefox-расширения. Меняем формат — правим **этот файл**, потом обе стороны (`WsServer.OnMessageAsync` + обработчики в `content.js`/`background.js`).

## Транспорт

- WebSocket поверх HTTP, без TLS.
- Адрес: `ws://localhost:17890/`
- Сервер — C# (`server-csharp/`), клиент — фоновый скрипт расширения (`extension-firefox/background.js`).
- Кодировка: UTF-8, текстовые WS-фреймы.

## Формат сообщения

Каждое сообщение — один JSON-объект:

```json
{ "type": "<string>", "payload": "<string>" }
```

Поля всегда строки. `payload` для сообщений без данных — пустая строка `""`.

## Сообщения: расширение → сервер

| type        | payload                    | смысл                                                                 |
|-------------|----------------------------|-----------------------------------------------------------------------|
| `hello`     | произвольная строка (инфо) | представление при подключении (необязательно)                         |
| `text`      | полный текст композера     | текст в `#prompt-textarea` стабилизировался — забери и вставь          |
| `recording` | `""`                       | запись реально началась (появилась кнопка `Submit`/`Cancel dictation`) |

- На `text` сервер сохраняет payload как «последний текст», отвечает `ack` и (если вооружён СТОПом) вставляет.
- На `recording` сервер возвращает фокус в окно, активное до старта диктовки.

## Сообщения: сервер → расширение

| type    | payload               | смысл                                                       |
|---------|-----------------------|-------------------------------------------------------------|
| `hello` | `"VoiceBridge ready"` | приветствие сразу после подключения                         |
| `ack`   | число (длина текста)  | подтверждение приёма `text`                                 |
| `mic`   | `""`                  | кликнуть «Start dictation» (старт), Ctrl+Win из `Idle`      |
| `stop`  | `""`                  | кликнуть «Submit dictation» (финал), Ctrl+Win из `Recording`|
| `clear` | `""`                  | очистить композер ChatGPT (после вставки)                   |

## Поток (happy path)

Управление — клавишей **Ctrl+Win** (ловится низкоуровневым хуком в сервере). Вариант **Ctrl+Win+«Y»** — то же завершение, но текст остаётся в буфере обмена.

1. Расширение подключается к `ws://localhost:17890/`, получает `hello`.
2. **Ctrl+Win** (`Idle`) → сервер запоминает активное окно, **на мгновение выносит окно Firefox на передний план** (иначе FF не стартует захват микрофона для фонового окна) → шлёт `mic` → расширение кликает «Start dictation».
3. Расширение, увидев индикатор записи (кнопка `Submit`/`Cancel dictation`), шлёт `recording` → сервер **возвращает фокус** в запомненное окно. Если сигнал не пришёл за `Config.RecordingFocusTimeoutMs` — фокус возвращается по страховочному таймеру.
4. Пользователь диктует (таб ChatGPT в фоне — запись уже идёт).
5. **Ctrl+Win** (`Recording`) → сервер запоминает активное окно (цель вставки) → шлёт `stop` → расширение кликает «Submit dictation».
6. ChatGPT транскрибирует → текст появляется в `#prompt-textarea` → content script шлёт `{type:"text", ...}`.
7. Сервер вооружён завершением → вставляет текст в запомненное окно (Ctrl+V) → шлёт `clear` (поле очищается под новый раунд).
   - Если был **Ctrl+Win+«Y»** — текст дополнительно остаётся в буфере обмена.

## Кнопки ChatGPT (для content script)

Якорь — стабильная `<form>` (класс содержит `composer`); контролы пересоздаются,
поэтому ищутся заново при каждом действии. Во время записи `#prompt-textarea` исчезает.

| действие              | селектор                                  |
|-----------------------|-------------------------------------------|
| микрофон (старт)      | `button[aria-label="Start dictation"]`    |
| птичка (финал)        | `button[aria-label="Submit dictation"]`   |
| отмена                | `button[aria-label="Cancel dictation"]`   |

Наличие кнопки `Submit`/`Cancel dictation` = запись идёт (индикатор для сигнала `recording`).
