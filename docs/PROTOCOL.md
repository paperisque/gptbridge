# Контракт WS — VoiceBridge ⇄ Firefox-расширение и сетевые клиенты

Единый источник правды для C#-сервера, Firefox-расширения и сетевого клиента. Меняем формат — правим **этот файл**, потом все стороны (`WsServer.OnMessageAsync` + `WsClient.OnMessage` + обработчики в `content.js`/`background.js`).

## Транспорт

- WebSocket поверх HTTP, без TLS.
- Адрес: `ws://<host>:17890/`, где `host`:
  - `localhost` — по умолчанию (всё на одной машине, без прав администратора);
  - `+` / IP машины-сервера — для доступа по сети (нужен разовый `netsh http add urlacl` и правило брандмауэра, см. README).
- Кодировка: UTF-8, текстовые WS-фреймы.

## Топология (сервер — хаб)

Сервер — центральный узел. К нему подключаются:

- **Firefox-расширение** (движок диктовки) — `background.js`. Кликает кнопки диктовки в ChatGPT, отдаёт распознанный текст.
- **Сетевые клиенты-контроллеры** — `VoiceBridge --connect <host>` на других машинах (`WsClient`/`ClientApp`). Ловят свой Ctrl+Win, захватывают своё активное окно, вставляют текст у себя.

Плюс **локальный контроллер** живёт в самом сервере (`ServerApp`) и по WS не ходит.

Сервер различает соединения по **роли**, определяемой из первого `hello`:
`payload` содержит `controller` → контроллер; иначе → Firefox.

**Владелец сессии.** Диктовка — один общий ресурс (один Firefox). Кто запустил сессию (локальный контроллер или конкретный сетевой клиент), тому сервер и вернёт распознанный текст. Пока идёт сессия одного владельца, запросы старта от других игнорируются («занято»).

## Формат сообщения

Каждое сообщение — один JSON-объект:

```json
{ "type": "<string>", "payload": "<string>" }
```

Поля всегда строки. `payload` для сообщений без данных — пустая строка `""`.

## Сообщения: Firefox-расширение → сервер

| type        | payload                    | смысл                                                                 |
|-------------|----------------------------|-----------------------------------------------------------------------|
| `hello`     | произвольная строка (инфо) | представление; payload БЕЗ слова `controller` ⇒ роль Firefox          |
| `text`      | полный текст композера     | текст в `#prompt-textarea` стабилизировался — забери и доставь владельцу |
| `recording` | `""`                       | запись реально началась (появилась кнопка `Submit`/`Cancel dictation`) |

- На `text` сервер сохраняет payload как «последний текст», отвечает `ack` и (если вооружён СТОПом) доставляет владельцу сессии.
- На `recording` сервер возвращает фокус в окно, активное до старта диктовки (на машине-сервере).

## Сообщения: сервер → Firefox-расширение

| type    | payload               | смысл                                                       |
|---------|-----------------------|-------------------------------------------------------------|
| `hello` | `"VoiceBridge ready"` | приветствие сразу после подключения                         |
| `ack`   | число (длина текста)  | подтверждение приёма `text`                                 |
| `mic`   | `""`                  | кликнуть «Start dictation» (старт)                          |
| `stop`  | `""`                  | кликнуть «Submit dictation» (финал)                         |
| `clear` | `""`                  | очистить композер ChatGPT (после доставки текста)           |

`mic`/`stop`/`clear` шлются адресно — только соединениям-Firefox (`SendToFirefox`).

## Сообщения: сетевой клиент → сервер

| type    | payload        | смысл                                                          |
|---------|----------------|----------------------------------------------------------------|
| `hello` | `"controller"` | представление; payload со словом `controller` ⇒ роль контроллер |
| `start` | `""`           | локальный Ctrl+Win из `Idle` — клиент просит старт диктовки     |
| `stop`  | `""`           | локальный Ctrl+Win из `Recording` — клиент просит финал         |

Клиент сам ловит хоткей, по `stop` захватывает СВОЁ активное окно и помнит вариант «+буфер» (Ctrl+Win+Y) — серверу это знать не нужно.

## Сообщения: сервер → сетевой клиент

| type     | payload                 | смысл                                                       |
|----------|-------------------------|-------------------------------------------------------------|
| `hello`  | `"VoiceBridge ready"`   | приветствие сразу после подключения                         |
| `inject` | текст диктовки          | этот текст вставить в окно, захваченное клиентом по `stop`   |

`inject` шлётся адресно владельцу сессии (`SendToController` по Id соединения).

## Поток (happy path)

Управление — клавишей **Ctrl+Win** (ловится низкоуровневым хуком: в сервере — для локального контроллера, в клиенте — для сетевого). Вариант **Ctrl+Win+«Y»** — то же завершение, но текст остаётся в буфере обмена (обрабатывается на стороне того контроллера, что вставляет).

1. Firefox-расширение подключается к серверу, шлёт `hello` (роль Firefox). Сетевые клиенты (если есть) подключаются и шлют `hello` с `controller`.
2. **Ctrl+Win** (`Idle`) у контроллера → сервер запоминает владельца сессии и активное окно (на машине-сервере), **на мгновение выносит окно Firefox на передний план** (иначе FF не стартует захват микрофона) → `mic` → расширение кликает «Start dictation».
   - локальный контроллер: жмёт хоткей сам сервер;
   - сетевой клиент: шлёт серверу `start`.
3. Расширение, увидев индикатор записи, шлёт `recording` → сервер **возвращает фокус** в запомненное окно (страховка по таймеру `Config.RecordingFocusTimeoutMs`).
4. Пользователь диктует (таб ChatGPT в фоне — запись уже идёт).
5. **Ctrl+Win** (`Recording`) у того же контроллера → `stop` → расширение кликает «Submit dictation». Окно-цель: для локального владельца захватывает сервер; для сетевого — сам клиент у себя.
6. ChatGPT транскрибирует → текст появляется в `#prompt-textarea` → content script шлёт `{type:"text", ...}`.
7. Сервер вооружён завершением → доставляет текст владельцу:
   - локальный владелец → вставка на месте (Ctrl+V);
   - сетевой владелец → `inject` ему по WS, клиент вставляет в своё окно.
   Затем сервер шлёт `clear` Firefox (поле очищается под новый раунд).
   - Если был **Ctrl+Win+«Y»** — текст дополнительно остаётся в буфере обмена той машины, где он вставлен.

## Кнопки ChatGPT (для content script)

Якорь — стабильная `<form>` (класс содержит `composer`); контролы пересоздаются,
поэтому ищутся заново при каждом действии. Во время записи `#prompt-textarea` исчезает.

| действие              | селектор                                  |
|-----------------------|-------------------------------------------|
| микрофон (старт)      | `button[aria-label="Start dictation"]`    |
| птичка (финал)        | `button[aria-label="Submit dictation"]`   |
| отмена                | `button[aria-label="Cancel dictation"]`   |

Наличие кнопки `Submit`/`Cancel dictation` = запись идёт (индикатор для сигнала `recording`).
