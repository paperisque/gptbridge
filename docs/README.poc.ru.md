# GPT Grabber &nbsp;<sub>[English version → poc-webview2/README.md](../poc-webview2/README.md)</sub>

**Автономный голосовой ввод: диктуешь в ChatGPT — текст по хоткею попадает в любое окно.**

GPT Grabber встраивает ChatGPT прямо через **WebView2** (Chromium), управляет его диктовкой, читает распознанный текст и вставляет его в то окно, где ты работаешь. В отличие от [основной версии на Firefox](README.ru.md), ей **не нужны ни внешний браузер, ни расширение, ни сетевой хаб** — это одно автономное приложение, живущее в трее.

## Хоткеи

| Хоткей | Действие |
|---|---|
| **Ctrl+Win** | Диктовка: старт → говори → стоп → вставка в активное окно |
| **Ctrl+Win+Y** | То же, но текст ещё и остаётся в буфере обмена |
| **Ctrl+Win+Alt** | Снова вставить последний распознанный текст |
| **Ctrl+Win** (в фазе подготовки / распознавания) | Отмена |

Тот же список — по кнопке **?** в окне приложения.

## Установка (для пользователей)

Собрать установщик одной командой (из корня репозитория):

```powershell
powershell -ExecutionPolicy Bypass -File tools\build-installer.ps1
```

Получится `dist\GptGrabber-Setup-<версия>.exe`. Установщик:

- ставит **для текущего пользователя, без прав администратора**, в `%LOCALAPPDATA%\GptGrabber`;
- кладёт рядом **портативный .NET 10 runtime** (приложение запускается через подписанный Microsoft `dotnet.exe`, поэтому Smart App Control его не блокирует — см. ниже);
- ставит **Microsoft Edge WebView2 Runtime**, если его нет (на Windows 11 он обычно уже есть);
- идёт **без шагов выбора** — запустил → поставилось → «Готово»; ярлыки в меню Пуск и на рабочем столе создаются сами.

При первом запуске войди в ChatGPT один раз прямо в окне приложения — сессия сохранится.

## Запуск из исходников (разработка)

Нужны **.NET 10 SDK** и **WebView2 Runtime**.

```powershell
cd poc-webview2
dotnet run
```

Готовый `.exe` не собирается (Windows Smart App Control блокирует неподписанный apphost), поэтому запуск всегда через доверенный `dotnet`. Для постоянного ярлыка на рабочем столе укажи `dotnet.exe`, а путь к DLL — аргументом.

### Флаги командной строки

Флаги идут **после** пути к DLL — `dotnet "…\WebView2Poc.dll" <флаги>`:

- `--lang en|de|ru` — язык интерфейса (перекрывает язык системы; по умолчанию — язык Windows);
- `--tray` — запуститься сразу свёрнутым в трей;
- `--no-beep` — выключить звук-подтверждение при вставке.

## Полезно знать

- **Живёт в трее.** Крестик (×) прячет окно в трей, а не закрывает; выход — через меню трея. Захват микрофона продолжается, пока окно скрыто или свёрнуто.
- **Один экземпляр.** Повторный запуск не создаёт вторую копию — разворачивает уже запущенную из трея.
- **Звук вставки.** Короткий нисходящий «дин-дон» подтверждает удачную вставку; выключается флагом `--no-beep`.
- **Чище ChatGPT.** Встроенная страница прячет лишнее вокруг композера (доп. кнопки и заголовок), чтобы не прыгало при сужении окна.
- **Где данные.** Профиль ChatGPT (WebView2) и лог лежат рядом с программой, в папке `data`: `%LOCALAPPDATA%\GptGrabber\data` для установленной версии, `bin\Debug\net10.0-windows\data` для сборки разработчика. Деинсталляция её удаляет.
- **Нет окна консоли.** Так как запуск идёт через консольный `dotnet.exe`, приложение на старте перезапускает себя без окна (возможно короткое мелькание); весь статус — иконка трея, «пилюля» поверх окон и лог-файл.
- **Иконка.** Перегенерировать `installer\app.ico` из `icons\ico2.png` — скриптом `tools\make-icon.ps1`.

Версия на Firefox — в основном [README.ru.md](README.ru.md); детали реализации и грабли исходного сервера — в [CLAUDE.md](../CLAUDE.md).
