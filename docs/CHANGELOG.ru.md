## 2026-06-11 — Надёжность старта/отбоя (по итогам теста)

- **«Запись идёт», а записи нет** (FF поднят вручную, таб ChatGPT из сессии, активен
  другой таб): прежний таб возвращался мгновенно по сигналу `recording`, который
  приходит РАНЬШЕ реального старта захвата, — уход с таба ChatGPT срывал захват.
  Теперь возврат прежнего таба отложен на 2.5 с (зеркально удержанию фокуса окна
  сервером) и отменяется при старте новой сессии.
- Прежний таб возвращается теперь ВСЕГДА — и когда таб ChatGPT создало само
  расширение (раньше в этом случае оставались в созданном табе). Прежний таб
  запоминается до любого переключения, ещё на этапе подготовки.

- **Два окна Firefox при холодном старте** — устранено: launch-grace (повторный запуск
  firefox.exe в течение 30 с подавляется; прогрев и старт записи больше не гоняются).
- **Долгое открытие таба ChatGPT** — ускорено: под запись таб создаётся АКТИВНЫМ (фоновые
  табы FF грузит лениво); ожидание «чужого» таба перед созданием своего сокращено до 0.8 с
  (но 12 с при только что поднятом FF — чтобы не плодить дубль его стартового таба).
  `ensureTab` получил payload-подсказки `start`/`warmup` + `,cold` (PROTOCOL.md).
- **Зависание на «Распознаю…», если захват не стартовал** — abort-путь: СТОП без
  подтверждённой записи шлёт `cancel` (клик «Cancel dictation») и сразу возвращает Idle;
  плюс страховка — текст не пришёл за 20 с после СТОПа → отбой вставки с ошибкой
  на индикаторе. Новое WS-сообщение `cancel` (сервер → расширение).
- **Запись с тишиной** (микрофон отвёрнут): распознаётся пустота, текста не будет —
  расширение это замечает и шлёт `empty`; сервер сразу даёт отбой («Пусто — текста
  нет»), не дожидаясь 20-секундного таймаута. Новое WS-сообщение `empty`.
  Исход определяется по КНОПКАМ композера, а не таймером: кнопка отправки
  (`#composer-submit-button`) = текст распознан; кнопка голосового режима на её месте =
  пусто. Таймер «пусто N секунд» давал ложное «Пусто» при медленном распознавании
  (текст приезжал позже и примешивался к следующей диктовке).
- Исправлен вечный `IsStartingUp` (переполнение при «ещё не запускали») — из-за него
  Firefox не стартовал сам, а подготовка таба всегда шла по медленному cold-пути.
- **Холодный старт с восстановлением сессии**: Firefox теперь запускается БЕЗ URL
  (URL давал дубль к табу из сессии), а «ленивый» (discarded) восстановленный таб
  ChatGPT расширение будит (активация/перезагрузка) — раньше оно молча ждало ответа
  от незагруженной страницы и старт срывался по таймауту.
- **Ctrl+Win вне пары «старт/стоп» = универсальная отмена**: во время подготовки —
  отмена старта; во время распознавания после стопа — отмена вставки (поздний текст
  не вставляется и вычищается из композера); при незапустившейся записи — отбой
  (как раньше). Индикатор показывает «Отменено».

## 2026-06-11 — Мультиязычность (en/de/ru)

- Все статичные надписи (консоль сервера/клиента + индикатор) — через `Lang.T` и
  JSON-модули `lang/{en,de,ru}.json`. По умолчанию — язык Windows
  (`GetUserDefaultUILanguage`, т.к. `CultureInfo` обезврежен `InvariantGlobalization`),
  флаг `--lang en|de|ru` перебивает. Фолбэк: выбранный язык → английский → ключ.

## 2026-06-11 — Системный индикатор статуса

- **Индикатор-«пилюля» поверх всех окон** (`StatusOverlay.cs`): видно из любого приложения,
  что происходит с диктовкой. Фазы: оранжевая звёздочка-мерцалка «Готовлю ChatGPT…» →
  оранжевый пульсирующий микрофон «Включаю микрофон…» → зелёный пульсирующий микрофон
  «Идёт запись» → синяя звёздочка «Распознаю…» → галочка «Вставлено» (гаснет сама).
  Ошибки — красный крестик.
- Layered-окно без WinForms/WPF: фокус не крадёт, клики проходят насквозь. Рендер —
  per-pixel alpha (`UpdateLayeredWindow`) + GDI+ со сглаживанием (System.Drawing.Common):
  гладкая капсула с белой рамкой. Значки — глифы Segoe MDL2 Assets, звёздочка —
  кадры-дингбаты; анимация — WM_TIMER.
- Якорь — в 3 px правее текстовой каретки активного окна; для Electron/Chromium (каретку
  не отдают) — низ-центр монитора, «как у Whisper».
- Белая рамка по контуру пилюли (читаемость на любом фоне).
- Флаги: `--overlay-compact` (только значок-кружок, без подписи), `--no-overlay` (выключить),
  `--overlay-colors key=RRGGBB,…` (перекрасить: bg, border, text, wait, rec, busy, err).
- Скоуп v1 — локальный пользователь сервера; сетевым клиентам индикатор не транслируется.
- `Injector.Inject` теперь возвращает успех/неуспех (для фаз «Вставлено»/«Вставка не удалась»).

## 2026-06-10 — Надёжный старт записи + dev-цикл

- **Главный фикс:** сервер держит фокус на окне Firefox ≥ 1.5 с после `mic`. Раньше фокус
  возвращался по сигналу `recording` (~11 мс после открытия UI диктовки), и Firefox не
  успевал реально включить микрофон — диктовка не стартовала.
- **Авто-подготовка Firefox/таба** (Idle→Preparing→Recording): сервер сам поднимает закрытый
  Firefox; расширение готовит таб по честной готовности (загрузка страницы + кнопка диктовки
  + осадка), без слепых задержек перед кликом. Устранён дубль таба ChatGPT при холодном старте.
- Стоп в созданном нами табе — без переключений «туда-сюда».
- **Dev-инструменты расширения:** `tools/update-ext.ps1` (обновление плагина одной командой
  + авто-перезагрузка таба), `tools/build-xpi.ps1`, `package.json` (web-ext lint/build/sign).
- Метки времени до миллисекунд в логах плагина и сервера.

## 2026-06-08 — Сетевой режим (хаб)

- Сервер стал хабом: один Firefox-движок диктовки + сетевые клиенты (`--connect <IP>`).

  (локальный Ctrl+Win + вставка на этой же машине).
