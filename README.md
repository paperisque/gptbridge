# Gptgraber

**Голосовой мост: диктовка ChatGPT → текст в любом окне, по хоткею.**

Диктуешь голосом, ChatGPT в фоновом табе Firefox расшифровывает речь — и текст по нажатию **Ctrl+Win** появляется прямо в окне, где ты работаешь (VS Code, редактор, мессенджер, что угодно). Переключаться в браузер не нужно. А по локальной сети текст можно получать и на другом компьютере.

## Польза

- **Качество ChatGPT, но везде.** Распознавание с пунктуацией и поддержкой многих языков — не только внутри чата, а в любом приложении.
- **Отдельный платный диктовщик не нужен** — задействуешь диктовку, которая уже встроена в ChatGPT (по духу — как WhisperFlow).
- **Не отрываешься от работы.** Триггер — один хоткей из любого приложения; фокус, на миг уйдя в браузер, сам возвращается к тебе.
- **Работает по сети.** Один компьютер с Firefox раздаёт диктовку нескольким машинам в локалке — текст прилетает в активное окно той машины, с которой ты нажал хоткей.

## Как это работает

Нажатие **Ctrl+Win** ловит низкоуровневый хук клавиатуры в C#-сервере. Сервер командует расширению кликнуть кнопку диктовки в ChatGPT; расширение читает расшифровку и шлёт её серверу по WebSocket; сервер доставляет текст в активное окно. Цель вставки — окно, которое было активным **в момент остановки** диктовки.

```
 ┌────────────┐   Ctrl+V (синтез)   ┌──────────────┐   mic/stop/clear   ┌────────────┐
 │ активное   │ ◄────────────────── │  VoiceBridge │ ─────────────────► │  Firefox   │
 │ окно (цель)│                     │   (сервер)   │ ◄───────────────── │  (ChatGPT) │
 └────────────┘                     └──────────────┘   text/recording   └────────────┘
```

### По сети: один Firefox — много машин

Сервер работает как **хаб**. Firefox-«движок диктовки» нужен только на машине с сервером. «Контроллеров» (тех, кто ловит хоткей и принимает текст в своё окно) может быть несколько: локальный встроен в сам сервер, а сетевые подключаются с других машин командой `VoiceBridge --connect <IP-сервера>`. Сетевой клиент ловит свой Ctrl+Win и шлёт серверу запрос; сервер гоняет общий Firefox, а распознанный текст возвращает **именно тому**, кто запустил диктовку, — для вставки в его активное окно.

```
   Ноутбук (сетевой клиент)            Десктоп (сервер + Firefox)
  ┌──────────────────────┐  start/stop ┌──────────────┐  mic/stop/clear  ┌────────────┐
  │ Ctrl+Win → запрос    │ ──────────► │  VoiceBridge │ ───────────────► │  Firefox   │
  │ вставка в своё окно  │             │    (хаб)     │ ◄─────────────── │  (ChatGPT) │
  └──────────────────────┘ ◄────────── └──────────────┘  text/recording  └────────────┘
                            inject(текст)
```

Диктовка — общий ресурс на один Firefox: пока идёт одна сессия, запросы от других контроллеров игнорируются («занято»). Защиты/токена нет — рассчитано на доверенную локальную сеть.

## Как пользоваться

### Локально, на одной машине

1. Запусти сервер и загрузи расширение (см. «Установка и запуск»). На кнопке расширения горит зелёная точка ● — значит мост на связи.
2. Работай в своём приложении. Когда нужно надиктовать:
   - **Ctrl+Win** — старт. Окно Firefox на миг мелькнёт (так браузер разрешает запись микрофона), фокус сразу вернётся к тебе. Говори.
   - **Ctrl+Win** ещё раз — стоп. Через 1–3 с распознанный текст вставится в твоё окно.
   - **Ctrl+Win+Y** — то же, что стоп, но текст ещё и останется в буфере обмена.

### С другого компьютера в локальной сети

1. На машине с Firefox запусти сервер в сетевом режиме, на клиентской — сетевой клиент (см. «Доступ по сети»).
2. На клиенте — те же хоткеи: **Ctrl+Win** старт/стоп, **Ctrl+Win+Y** с сохранением в буфер. Firefox при старте мелькнёт **на сервере** (ты его не видишь), а текст прилетит в активное окно **твоей** машины.

## Установка и запуск

Нужен **.NET 10 SDK**. Готовый `.exe` проект не собирает: его блокирует Windows Smart App Control (SAC), поэтому запуск всегда через `dotnet` (он подписан и доверенный). Справка по аргументам — `dotnet run -- --help`.

### Сервер

```powershell
cd server-csharp\VoiceBridge
dotnet run
```

Консольное окно и есть сервер (там же логи), `Ctrl+C` — выход. По умолчанию слушает `localhost:17890` — это работает без прав администратора. Если порт не открывается с ошибкой доступа, выполни один раз в админ-консоли:

```powershell
netsh http add urlacl url=http://localhost:17890/ user=ДОМЕН\Пользователь
```

### Расширение Firefox

`about:debugging#/runtime/this-firefox` → **Load Temporary Add-on…** → выбери `extension-firefox/manifest.json`. Зелёная точка ● на кнопке расширения = WebSocket подключён к серверу. Подробнее — в [extension-firefox/README.md](extension-firefox/README.md).

### Доступ по сети (только если сильно надо)

**На машине-сервере** (где Firefox) запусти сервер на всех интерфейсах:

```powershell
dotnet run -- --host +
```

и разово открой доступ (админ-консоль): резервация URL + правило брандмауэра **только для локальной подсети** (наружу, в интернет, порт при этом не открывается — машина за NAT с приватным IP):

```powershell
netsh http add urlacl url=http://+:17890/ user=ДОМЕН\Пользователь
netsh advfirewall firewall add rule name="VoiceBridge 17890" dir=in action=allow protocol=TCP localport=17890 remoteip=localsubnet
```

**На клиентской машине** (Firefox и расширение там не нужны — только .NET):

```powershell
dotnet run -- --connect 192.168.1.50    # IP машины-сервера
```

Порт по умолчанию `17890`; сменить — флагом `--port` (тогда поправь его и в расширении: `extension-firefox/background.js` и `manifest.json`).

## Что под капотом

- **Управление.** Низкоуровневый хук клавиатуры (`WH_KEYBOARD_LL`) ловит Ctrl+Win (это «модификатор-тоггл», обычный `RegisterHotKey` так не умеет). Клавиша «+буфер» опознаётся по **скан-коду**, поэтому не зависит от раскладки.
- **Старт записи.** Firefox не пишет микрофон, пока его окно в фоне. Поэтому сервер на миг выносит окно браузера на передний план, запись стартует, и фокус тут же возвращается в рабочее окно.
- **Вставка.** Программа кладёт текст в буфер обмена и **сама** синтезирует нажатие Ctrl+V через `SendInput` (скан-коды) — ты ничего не нажимаешь. Это единственный способ доставить текст в поля Electron/Chromium (например, VS Code), и он корректно вставляет любой Unicode.
- **Связь.** WebSocket на порту `17890`. Сервер — хаб: различает Firefox-расширение и сетевых клиентов, помнит владельца текущей сессии и возвращает текст диктовки именно тому, кто её запустил.
- **Запуск.** Один проект — два режима: без аргументов это сервер, с `--connect <IP>` — сетевой клиент. Оба переиспользуют один и тот же код вставки и хука.

Детали реализации и грабли — в [CLAUDE.md](CLAUDE.md), полный контракт WebSocket-сообщений — в [docs/PROTOCOL.md](docs/PROTOCOL.md).
