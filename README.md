# Gptgraber

**Голосовой мост: диктовка ChatGPT → системный ввод куда угодно.**

Диктуешь в табе ChatGPT (Firefox), а текст одной клавишей вставляется в то окно, где ты сейчас работаешь — VS Code, редактор, мессенджер. Идея — использовать качественную диктовку ChatGPT как системный голосовой ввод, не переключаясь в браузер.

Управление — в стиле WhisperFlow, одной клавишей. Браузер живёт в фоне; расширение само кликает кнопки диктовки, читает результат и отдаёт серверу, сервер вставляет текст в активное окно.

```
┌─────────────┐   WS localhost:17890   ┌──────────────────┐   Ctrl+V (SendInput)   ┌────────────┐
│  Firefox    │ ◄────────────────────► │  C#-сервер       │ ─────────────────────► │ активное   │
│  расширение │  mic/stop/clear ▲ ▼ text│  VoiceBridge     │                        │ окно (цель)│
│  (ChatGPT)  │                         │  хук + инъекция  │                        │            │
└─────────────┘                         └──────────────────┘                        └────────────┘
```

## Как пользоваться

1. Запусти сервер и загрузи расширение (см. ниже). На кнопке расширения — зелёная точка ●.
2. Работай в своём приложении. Когда нужно надиктовать:
   - **Ctrl+Win** — старт. Окно Firefox на миг мелькнёт (так Firefox разрешает запись), фокус сам вернётся к тебе. Говори.
   - **Ctrl+Win** ещё раз — стоп. Через 1–3 с распознанный текст вставится в твоё окно.
   - **Ctrl+Win+Y** — то же, но текст ещё и останется в буфере обмена.

## Сборка и запуск

### Сервер (`server-csharp`)

Нужен **.NET 10 SDK**.

```powershell
cd server-csharp\VoiceBridge
dotnet run
```

Окно консоли — это и есть сервер (там же логи). `Ctrl+C` — выход.

> **Права на порт.** `http://localhost:17890/` обычно открывается без админ-прав. Если `dotnet run` ругается отказом доступа — выполни один раз в админ-консоли:
> ```
> netsh http add urlacl url=http://localhost:17890/ user=ДОМЕН\Пользователь
> ```

### Расширение (`extension-firefox`)

Firefox (Developer Edition удобнее):

1. `about:debugging#/runtime/this-firefox` → **Load Temporary Add-on…** → выбери `extension-firefox/manifest.json`.
2. На кнопке расширения появится зелёная точка ● — WS подключён. Красная/пусто — сервер не запущен (расширение само переподключится, когда сервер поднимется).

Подробнее — в [extension-firefox/README.md](extension-firefox/README.md). Временное дополнение слетает при перезапуске браузера — для разработки это норма.

## Как это устроено

### Управление одной клавишей

Хоткеи ловит низкоуровневый хук клавиатуры (`WH_KEYBOARD_LL`) в C#-сервере — `RegisterHotKey` не умеет «только модификаторы» (Ctrl+Win). Жест распознаётся на отпускании комбо. Клавиша варианта «+буфер» ловится по **скан-коду** (позиция клавиши), а не по виртуальному коду — иначе она ломалась бы при смене раскладки (немецкая/русская дают разные vkCode для одной клавиши).

### Firefox и микрофон в фоне

Firefox **не стартует захват микрофона, пока окно в фоне** (защита от скрытой записи). Поэтому на старте сервер находит окно Firefox с ChatGPT, на мгновение выносит его на передний план — запись стартует — и сразу возвращает фокус в твоё рабочее окно. Раз начавшись, запись спокойно продолжается в фоне.

### Вставка в чужое окно

Цель часто Electron/Chromium (VS Code): текстовое поле там — виджет внутри рендерера, а не Win32 `EDIT`, и `WM_PASTE`/`WM_CHAR`/`WM_SETTEXT` до него не доходят. Зато он реагирует на **настоящий ввод** через системную очередь. Поэтому текст кладётся в буфер обмена и вставляется синтезом `Ctrl+V` через `SendInput` скан-кодами — для приложения это неотличимо от живой клавиатуры, и любой Unicode (кириллица) вставляется атомарно. Прежнее содержимое буфера восстанавливается (кроме варианта Ctrl+Win+Y).

### Обход защиты от «кражи фокуса»

`SetForegroundWindow` срабатывает, только если процесс уже на переднем плане. Сервер фоновый, поэтому `Injector.ForceForeground` использует обход: снять `SPI_SETFOREGROUNDLOCKTIMEOUT` → `AttachThreadInput` (склеить очереди ввода с foreground- и target-потоком) → `SetForegroundWindow`/`SetFocus` → обязательно отвязаться и вернуть таймаут. Тем же механизмом окно Firefox поднимается на старте.

### Почему расширение, а не userscript

WebSocket до `localhost` открывается из background-контекста расширения, который **не подчиняется CSP страницы ChatGPT**. Userscript (Tampermonkey) исполняется в контексте страницы — его WS зарезал бы `connect-src` из CSP. Background — persistent MV2 (MV3 event-page усыпил бы соединение).
